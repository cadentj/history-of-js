# Pain 3: reproducible dependencies

This folder captures the "solution" side of the lesson where we move from manually-vendored scripts to a package manifest.

- `package.json` pins first-party dependency intent.
- `package-lock.json` is generated and locks the full tree.

Code between `BEGIN_PROBLEM_EXCLUDE` and `END_PROBLEM_EXCLUDE` is removed when generating the paired `problem/` directory.
